上一章已经讲了如何搭建开发环境,如果你还不清楚如何搭建环境请先移步搭建开发环境，现在就来使用RN开发出你的第一个Hello Wrold吧。

  • 首先初始化一个项目
    打开命令行窗口，进入我们希望建立的项目文件的父目录:
    (如 cd /Users/file/Desktop/reactnative)
    进入父目录后输入命令：

    react-native init ReactNative

其中ReactNative表示你希望建立的工程名，第一次建立会比较慢，慢的程度取决于你网速，在mac下会更慢，因为他会同时下载android和ios平台的所需数据，请耐心等待。
当出现如下图所示，则表示初始化成功，

大家可以去刚才指定的文件目录看到已经初始化完成的项目文件，项目结构如下图所示，分别有android和ios工程文件，然后使用命令行窗口进入此目录下cd ReactNative,然后执行升级命令：

react-native upgrade

接下来就我们就把这个RN项目运行起来了，android的话建议大家使用android手机调试，IOS的话用自带的模拟器。

####运行在android平台
首先把数据线连接上电脑和你的android手机，手机请打开开发者模式的usb调试。然后在命令行输入命令:adb device 如果手机连接正确，会看到如下显示：

list of devices attached
637b3a75 device

如没有显示，则表示没有连接成功，无法调试，如果你的测试机是android5.0以上的系统，还需要输入如下命令：

adb reverse tcp:8081 tcp:8081

这条命令是通过adb反向代理端口，将电脑的8081端口反向代理到测试机上。
如果测试是5.0一下则不需要执行以上命令。

然后在安装目录下输入命令

react-native run-android

此命令会编译android项目并将安装包安装到你的测试机中，此命令第一次输入的时间可能会比较长，之后就会很快了。
如果你使用的是Windows系统，程序安装好之后在手机上会显示红色屏幕，这是因为电脑上的服务环境还没有启动，请在在命令行输入命令：

react-native start

如果使用的测试机或模拟器低于5.0版本也会显示红色屏幕，按下menu键或者摇动手机，会在屏幕上弹出菜单，然后选择”Dev Setting”，在选择”Debug server host&port for device”，在弹出框中输入当前电脑的ip地址,（如不知道地址在命令行输入ipconfig可以找到)，如当前IP为192.168.1.5，就在输入框中输入192.168.1.5:8081点击确定返回红色界面在点击Reload JavaScript按钮，这时界面就应该出来了。
如果运行中出现如下图错误，则需要在android目录下加入local.properties文件，如果你有原来运行过的android工程则可以直接拷贝过来就行了。
如果没有则自己创建一个,在里面输入：
sdk.dir=你的android SDK路径 如:(/Users/koala/Library/Android/sdk)

  • 注：android5.0以上的手机每次运行都需要输入adb reverse tcp:8081 tcp:8081命令和react-native start命令
